# 交易模組 v1 — 店長驗收清單

這份是給 **店長／工讀生使用** 的驗收清單，不是工程師用。
把每一個情境照著做一次，確認系統跟你的預期一樣。

---

## 開始之前

- 請用 **店長身份** 登入後台
- 找一位「測試用顧客」，最好是還沒實際消費過的
- 進入：**後台 → 交易紀錄**

---

## 一、看得到新功能

進入交易紀錄頁面後，請確認：

- [ ] 表格多了一欄「**狀態**」，每筆都有顯示狀態小標籤（已完成 / 已作廢 / 已取消 / 已退款）
- [ ] 表格最右邊多了一欄「**操作**」，每一列右側有一個「**⋯**」小按鈕

如果上面看不到，先重整頁面。還是看不到 → 請聯絡技術人員，**不要繼續往下做**。

---

## 二、查看交易詳情

- [ ] 隨便挑一筆交易，點右側的「⋯」按鈕
- [ ] 畫面右側應該滑出一個側邊抽屜（**Drawer**）
- [ ] 抽屜裡可以看到：
  - 顧客姓名
  - 金額
  - 付款方式
  - 歸屬店長
  - 交易日期
- [ ] 如果這筆是「課程購買」，還會看到「**影響內容**」區塊：方案名稱、總堂數、已使用 / 已預約 / 可用 / 作廢數量
- [ ] 抽屜底部有「**異動紀錄**」區塊（一開始可能是「尚無異動」）

關閉抽屜：點空白處 / 點右上角 ✕ / 按鍵盤 Esc。

---

## 三、修改備註（不需要原因）

- [ ] 開一筆交易的詳情抽屜
- [ ] 在「備註」區塊點「修改」
- [ ] 輸入任意文字，例如「測試備註 2026/04/30」，點「儲存」
- [ ] 儲存後，列表「備註」欄應該立刻看到新內容
- [ ] 重新打開抽屜 → 「異動紀錄」應該看到一筆「**修改備註**」的紀錄

---

## 四、修改付款方式（**必須填原因**）

- [ ] 開一筆交易的詳情
- [ ] 在「更正付款方式」區塊點「變更」
- [ ] 選一個跟原本不同的付款方式
- [ ] **不填原因** → 確認「儲存」按鈕是 **灰色不能按**
- [ ] 填入原因，例如「顧客原本說現金，後來改匯款」
- [ ] 「儲存」按鈕應該變得可以按 → 點下去
- [ ] 列表的「付款方式」欄應該立刻更新
- [ ] 重新打開抽屜 → 「異動紀錄」會看到一筆「**更正付款方式**」紀錄，並包含你填的原因

---

## 五、修改歸屬店長（**必須填原因**）

- [ ] 開一筆交易的詳情
- [ ] 在「更正歸屬店長」區塊點「變更」
- [ ] 選一個不同的店長
- [ ] 不填原因 → 「儲存」按鈕應該按不下去
- [ ] 填入原因，例如「實際成交是另一位店長」
- [ ] 點「儲存」
- [ ] 列表的「歸屬店長」欄應該立刻更新
- [ ] 「異動紀錄」會看到一筆「**更正歸屬店長**」紀錄

---

## 六、取消未使用的課程購買（最關鍵情境 ✨）

> 這個情境必須驗證 — 取消後顧客方案會跟著作廢

**準備：** 找一筆「課程購買」交易，且該方案 **完全沒被預約過**。
（影響內容區塊應該顯示：已使用 0 / 已預約 0 / 可用 N / 作廢 0）

步驟：

- [ ] 開該筆交易的詳情抽屜
- [ ] 拉到底部的「**危險操作**」區塊
- [ ] 點「**取消交易**」
- [ ] 抽屜內畫面應該切換到「確認取消交易」區塊（**不會跳出彈窗**）
- [ ] 看到的內容應該是：
  - 取消後將同步的事項條列（營收扣除、堂數扣回、前台同步、異動紀錄）
  - 一個原因輸入框（必填）
  - 「返回」與「確認取消交易」兩顆按鈕
- [ ] 不填原因 → 「確認取消交易」按鈕灰色按不下去
- [ ] 填入原因，例如「顧客匯錯金額，重新建單」
- [ ] 點「**確認取消交易**」

完成後請驗證：

- [ ] 抽屜回到主畫面，狀態變成「**已作廢**」
- [ ] 編輯區塊（備註 / 付款方式 / 歸屬店長 / 危險操作）**全部消失**
- [ ] 多了一個「**作廢資訊**」區塊，顯示作廢時間 / 作廢人 / 原因
- [ ] 異動紀錄多一筆「**取消交易**」
- [ ] 回到列表，這列變灰色 + 顧客名字 / 金額有刪除線
- [ ] 「本頁收入合計」應該 **少了這筆金額**
- [ ] 進入該顧客的方案頁面 → 該方案應該消失或標示為已取消
- [ ] 從顧客手機 LINE 端進「我的方案」→ 該方案不在可用列表

---

## 七、嘗試取消「已有預約」的方案（應該擋下來）

**準備：** 找一筆課程購買交易，方案有至少一筆預約佔用堂數。

- [ ] 開詳情，拉到底「危險操作」
- [ ] 點「取消交易」
- [ ] 填原因，點「確認取消交易」
- [ ] 應該 **失敗**，跳出紅色錯誤訊息：
  > 「此方案已有預約佔用堂數，請先取消相關預約後，再取消交易。」
- [ ] 交易仍然是「已完成」狀態（**沒有被改動**）
- [ ] 顧客方案、預約、堂數通通沒變

---

## 八、嘗試取消「已有完成堂數」的方案（應該擋下來）

**準備：** 找一筆課程購買交易，方案至少有一堂顯示為「已完成」。

- [ ] 同上流程，點「取消交易」+ 填原因 + 確認
- [ ] 應該 **失敗**：
  > 「此方案已有完成堂數，不能直接取消交易。如需退款，請使用退款流程。」
- [ ] 交易、方案、堂數通通沒變

---

## 九、取消「單次消費」交易

- [ ] 找一筆「單次消費」或「體驗購買」交易
- [ ] 點「⋯」→「取消交易」→ 填原因 → 確認
- [ ] 應該成功（單次交易沒有方案綁定，不會被擋）
- [ ] 列表灰化 + 本頁收入扣除

---

## 十、作廢交易不可再編輯（防呆）

- [ ] 任意打開一筆已作廢交易
- [ ] 確認 **看不到** 任何編輯按鈕（修改備註 / 變更付款方式 / 變更歸屬店長 / 取消交易）
- [ ] 只能查看「作廢資訊」與「異動紀錄」

---

## 十一、營收交叉驗證（重點 ✨）

取消交易**前後**做以下對照，數字要一致：

| 來源 | 取消前金額 | 取消後金額 | 應差距 |
|---|---|---|---|
| 後台首頁「今日營收」卡片 | | | 等於那筆金額 |
| 後台首頁「本月營收」卡片 | | | 等於那筆金額 |
| 報表頁「本月營收」 | | | 等於那筆金額 |
| 交易紀錄頁「本頁收入合計」 | | | 等於那筆金額 |
| 匯出 CSV「總計」 | | | 等於那筆金額 |

**任何一個對不上 → 立刻回報，先停止使用。**

---

## 十二、合作店長權限驗證（如果你有合作店長帳號）

- [ ] 用合作店長身份登入
- [ ] 開一筆交易詳情
- [ ] 應該 **看不到**：「更正付款方式」、「更正歸屬店長」、「取消交易」按鈕
- [ ] **應該看得到**：「修改備註」（合作店長可以改備註）

如果合作店長看到了「取消交易」按鈕 → 立刻回報。

## 異常情境處理

### Drawer 打開後一直顯示「載入中…」或紅色錯誤
1. 重整頁面再試一次
2. 看到「資料表結構尚未更新，請先執行 migration 後再使用此功能」→ **這是技術人員需要處理的事**，請聯繫技術人員，**不要繼續使用編輯功能**
3. 看到其他錯誤 → 截圖回報

### 取消交易後，顧客方案還沒同步
1. 重整顧客方案頁
2. 從顧客手機端 LINE 重新進「我的方案」
3. 還是看到 → 截圖回報

### 修改了某個欄位，列表沒更新
1. 重整列表
2. 還是沒更新 → 截圖回報

---

## 完成驗收後

- 所有勾選都打勾、營收交叉驗證全部一致 → ✅ 可以正式上線
- 任何一項對不上 → 暫停上線，回報技術人員

## 給技術人員的對應資訊

驗收清單對應的功能模組：
- 規格：交易模組 v1 — 安全修正與取消交易
- Migration：`prisma/migrations/20260429_add_transaction_void_and_audit_log/`
- Server actions：`updateTransactionNote / updateTransactionPaymentMethod / updateTransactionOwnerStaff / voidTransaction`
- UI：`src/app/(dashboard)/dashboard/transactions/_components/`
- 權限 key：`transaction.void`（OWNER 預設有，PARTNER 預設無）
- 部署順序：(1) 跑 migration → (2) 跑 permission backfill SQL → (3) 失效 staff-permissions cache → (4) 開放店長使用
