health-tracker Part A — 身體數據系統 API 建立

在現有 health-tracker Next.js 專案中新增 3 個 API Route + 1 個認證 helper。不改動任何現有功能。

A1 建立 API Key 認證 helper

新建 src/lib/api-auth.ts

  • 從 request header 讀取 X-API-Key
  • 與 env INTERNAL_API_KEY 比對
  • 不符合 → 回傳 401 JSON
  • 加入簡易 rate limit（optional，可用 Map 記錄 IP + timestamp）
src/lib/api-auth.ts 範例
// API Key 驗證，用於 server-to-server 呼叫
import { NextRequest, NextResponse } from "next/server";

export function validateApiKey(req: NextRequest): NextResponse | null {
  const apiKey = req.headers.get("x-api-key");
  if (!apiKey || apiKey !== process.env.INTERNAL_API_KEY) {
    return NextResponse.json(
      { error: "Unauthorized" },
      { status: 401 }
    );
  }
  return null; // null = 驗證通過
}
.env.local 新增：INTERNAL_API_KEY=（隨機產生一組 32+ 字元的 key，蒸足系統也要存一份）
A2 API: 以 email 查詢 profile

新建 src/app/api/health/profile/route.ts

  • Method: GET
  • Query: ?email=xxx@gmail.com
  • 用 Supabase server client 查 profiles 表（join auth.users 的 email）
  • 找到 → 回傳 { found: true, profile: { id, fullName, gender, age, height } }
  • 找不到 → 回傳 { found: false }
  • 不回傳任何 auth token / password / session 資料
GET /api/health/profile?email=wang@gmail.com
// Response 200
{
  "found": true,
  "profile": {
    "id": "a1b2c3d4-...",      // profiles.id (UUID)
    "fullName": "王小明",
    "gender": "male",           // "male" | "female" | null
    "age": 35,
    "height": 175
  }
}

// 找不到時
{ "found": false }
注意：health-tracker 的 profiles 表用 auth.users.id 作為 id，email 存在 auth.users 表。查詢時需要先從 Supabase Auth admin API 或用 service_role key 查 auth.users.email → 取得 user.id → 再查 profiles。或者如果 profiles 表有存 email 欄位就直接查。請先確認 profiles 表是否有 email 欄位。

A4 驗收：用 curl 測試 API

部署後用以下指令驗收：

# 1. 無 API Key → 應回 401
curl https://health-tracker-eight-rosy.vercel.app/api/health/profile?email=test@gmail.com

# 2. 有 API Key + 存在的 email → 應回 profile
curl -H "X-API-Key: YOUR_KEY" \
  "https://health-tracker-eight-rosy.vercel.app/api/health/profile?email=真實email"

# 3. 有 API Key + 不存在的 email → 應回 { found: false }
curl -H "X-API-Key: YOUR_KEY" \
  "https://health-tracker-eight-rosy.vercel.app/api/health/profile?email=nobody@xxx.com"

# 4. Summary → 應回完整摘要
curl -H "X-API-Key: YOUR_KEY" \
  "https://health-tracker-eight-rosy.vercel.app/api/health/summary?profileId=取得的UUID"

B3 Query: 自動比對 + 綁定邏輯
