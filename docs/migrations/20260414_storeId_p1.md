# P1 storeId 多店隔離 Migration 說明

## 1. 本次 Migration 列表

| Migration | 對應資料表 |
|-----------|-----------|
| `20260414_add_store_to_misc_tables` | MakeupCredit, SpaceFeeRecord, MessageLog |
| `20260414_add_store_to_ops_reconciliation` | OpsActionLog, ReconciliationRun |
| `20260414_add_store_to_reminder_template` | ReminderRule, MessageTemplate |

共計 **7 張表**新增 storeId 欄位。

## 2. 執行順序

```
1. 20260414_add_store_to_misc_tables        (先)
2. 20260414_add_store_to_ops_reconciliation  (次)
3. 20260414_add_store_to_reminder_template   (末)
```

三份之間**無相依關係**，各自獨立操作不同的表。執行順序由 Prisma 按字母排序自動決定，不影響結果。

## 3. 每份 Migration 的作用

### 3.1 `20260414_add_store_to_misc_tables`

| 表 | 操作 |
|---|------|
| MakeupCredit | 加 storeId，從 Customer.storeId 推導 backfill |
| SpaceFeeRecord | 加 storeId，從 Staff.storeId 推導 backfill |
| MessageLog | 加 storeId，從 Customer.storeId 推導 backfill |

- 無法推導者 fallback 到 `default-store`
- 加 NOT NULL + FK + @@index([storeId])
- 無資料被停用/改名/刪除

### 3.2 `20260414_add_store_to_ops_reconciliation`

| 表 | 操作 |
|---|------|
| OpsActionLog | 加 storeId，從 assigneeStaffId → Staff.storeId 推導，fallback default-store |
| ReconciliationRun | 加 storeId，全部設為 default-store |

- OpsActionLog 的 `@@unique([module, refId])` 改為 `@@unique([storeId, module, refId])`（舊 constraint 先 DROP 再建新的）
- 加 NOT NULL + FK + @@index([storeId])

### 3.3 `20260414_add_store_to_reminder_template`

| 表 | 操作 |
|---|------|
| ReminderRule | 加 storeId，backfill default-store，clone 給 demo-store（含 templateId 重新映射） |
| MessageTemplate | 加 storeId，backfill default-store，clone 給 demo-store |

- 加 NOT NULL + FK + @@index([storeId])
- demo-store 需要獨立的模板與規則，因此做了 clone

## 4. 為什麼是 3 份而不是 1 份

- P1 分三個模組平行開發（通知模板 / 營運追蹤+對帳 / 其他營運資料）
- 三組表之間無 FK 或邏輯相依，可安全並行
- Deploy 前已確認：
  - schema.prisma 的 Store model relations 無衝突（三組各自新增不同 relation）
  - TypeScript 全專案零錯誤
  - Prisma generate 成功
- 結果等價於單一 migration，差別僅在拆成三份記錄

## 5. 未來原則

- 後續 migration 儘量避免平行修改 schema
- 若平行開發，需在 deploy 前統一合併為單一 migration
- 已 deploy 的 migration 不可修改，只能新增增量 migration
- Migration 順序不可交錯，不可手動修改 `_prisma_migrations` 表
