# 最終驗收清單

> 最後更新：2026-04-06
> 適用版本：穩定模組 v1.2+
> 用途：每次 deploy 前 / 後快速驗收

## 優先級定義

| 等級 | 定義 | 失敗影響 |
|------|------|---------|
| **P0** | 數據正確性（報表、堂數、營收、權限、時區） | 帳務錯誤、資料外洩、金額不符 |
| **P1** | 功能完整性（流程可走通、操作有回應） | 功能不可用、流程中斷 |
| **P2** | UI / 體驗（顯示、排序、顏色、空狀態） | 不影響數據，但影響使用體驗 |

## 快速對帳流程（5 分鐘完整對帳）

> 每次 deploy 後用 Owner 帳號執行，確認所有營收來源數字一致

### Step 1：記錄基準值

1. 開 `/dashboard` → 記下「本月營收」卡片金額 → 記為 **A**
2. 開 `/dashboard` → 記下「今日營收」卡片金額 → 記為 **B**

### Step 2：報表頁交叉比對

3. 開 `/dashboard/reports` → 選「本月」preset
4. 記下「課程總收入」→ 記為 **C**，應 = **A**
5. 記下「退款」→ 記為 **D**
6. 記下「淨收入」→ 應 = **C** + **D**（D 為負數）
7. 記下「完成服務」堂數 → 記為 **E**
8. 切換 preset 為「今日」→ 記下「課程總收入」→ 應 = **B**

### Step 3：CSV 交叉比對

9. 回報表頁（本月）→ 點「全店月報 CSV」下載
10. 開 CSV → 找「合計」列 →「課程總收」應 = **A** = **C**
11. CSV「淨收」合計應 = Step 6 的淨收入
12. CSV「完成堂數」合計應 = **E**

### Step 4：交易紀錄比對

13. 開 `/dashboard/transactions` → 確認篩選為本月
14. 逐頁查看「本頁收入合計」（僅含 REVENUE_TYPES）
15. 若單頁資料量 ≤ 30 筆，本頁合計即為本月總營收，應 = **A**
16. 若多頁，可加總各頁合計或信任 Step 2~3 已驗證

### Step 5：店長對帳（可選）

17. 報表頁「店長明細」→ 選一位店長記下淨收 → 記為 **F**
18. 報表頁「收入類型」表 → 同一店長淨收 → 應 = **F**
19. 點「店長月報 CSV」→ 同一店長淨收 → 應 = **F**

### 判定

- **A = C = CSV合計 = 交易加總** → ✅ 月營收正確
- **B = 報表今日** → ✅ 日營收正確
- **淨收三源一致** → ✅ 退款計算正確
- **完成堂數兩源一致** → ✅ 堂數統計正確
- **店長三源一致** → ✅ 個人歸屬正確
- 以上任一不符 → ❌ P0 bug，立即排查

## 附錄：指標計算公式速查

| 指標 | 公式 | 排除 |
|------|------|------|
| 剩餘堂數 | `totalSessions - SUM(COMPLETED+NO_SHOW peoples) - SUM(PENDING+CONFIRMED peoples)` | `isMakeup=true`, `CANCELLED` |
| 今日預約筆數 | `COUNT(id) WHERE today AND (PENDING\|CONFIRMED)` | COMPLETED, NO_SHOW, CANCELLED |
| 今日預約人數 | `SUM(people) WHERE today AND (PENDING\|CONFIRMED)` | 同上 |
| 今日已完成 | `SUM(people) WHERE today AND COMPLETED` | — |
| 今日營收 | `SUM(amount) WHERE today AND REVENUE_TYPES` | REFUND, SESSION_DEDUCTION, ADJUSTMENT |
| 本月營收 | `SUM(amount) WHERE monthStart AND REVENUE_TYPES` | 同上 |
| 淨收入 | `課程總收入 + 退款`（退款為負數） | SESSION_DEDUCTION, ADJUSTMENT |
| 完成堂數(報表) | `COUNT(booking.id) WHERE COMPLETED`（筆數非人數） | CANCELLED |
| REVENUE_TYPES | `TRIAL_PURCHASE, SINGLE_PURCHASE, PACKAGE_PURCHASE, SUPPLEMENT` | — |
