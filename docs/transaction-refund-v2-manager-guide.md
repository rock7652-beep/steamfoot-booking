# 退款操作指南（給店長 / 工讀生）

這份說明退款功能怎麼用、什麼時候用、用完之後會發生什麼。
不是工程文件，是給「實際操作的人」看的。

---

## 一、先搞清楚：「取消交易」 vs 「退款」

兩個都會讓帳變少，但用途不一樣。

| 情境 | 用哪個 |
|---|---|
| 顧客買了之後反悔，**完全還沒用過、也沒預約** | 取消交易 |
| 顧客已經用過幾堂，剩餘堂數想退 | **退款** |
| 顧客用完一些，剩下不想用了，要拿回剩餘金額 | **退款** |
| 系統測試 / 誤建單 | 取消交易 |
| 顧客要拿錢回去（任何金額） | **退款** |

**簡單記法**：
- 「**這筆交易整個不算了**」→ 取消交易
- 「**這筆交易要還顧客錢**」→ 退款

---

## 二、退款怎麼操作

進入：**後台 → 交易紀錄 → 找到那筆「課程購買」 → 點 ⋯ → Drawer 打開**

往下拉，會看到「**退款**」區塊（橘色按鈕）。

> 注意：只有「**課程購買**」才會出現「退款」按鈕。
> 體驗、單次、補差額這些 v2 還不支援退款流程，需要走人工流程。

### 步驟

1. 點 [退款] → Drawer 切換到退款確認區
2. 選擇退款方式（兩種，下面會說明）
3. 看試算金額對不對
4. 填**退款原因**（一定要填）
5. 點 [確認退款]
6. 完成後 Drawer 自動回到主畫面，異動紀錄會多一筆

---

## 三、退款方式有兩種

### 方式 A — 全額退款（未使用方案）

**適用**：顧客**完全沒用過**這個方案。

**例**：賣 3 堂 NT$ 8,980，顧客還沒來、也沒預約 → 全額退 NT$ 8,980。

試算 panel 顯示：

```
原購買金額  NT$ 8,980
總堂數      3
已使用      0
預約佔用    0
可退款堂數  3
預計退款    NT$ 8,980
```

退款完成後：
- 顧客錢包 → 取消（不能再用）
- 該方案的所有堂數 → 標記為作廢
- 報表淨收入扣除 8,980
- 顧客手機端「我的方案」這個方案會消失

---

### 方式 B — 退剩餘堂數

**適用**：顧客**已經用過幾堂**，剩下不想用了。

**例**：賣 3 堂 NT$ 8,980，顧客來 1 堂之後不想再來。
- 單堂價：8,980 ÷ 3 ≈ 2,993
- 可退款：2,993 × 2（剩下 2 堂） = NT$ 5,987
- 已用過的 1 堂保留紀錄，**不會被影響**

試算 panel 顯示：

```
原購買金額  NT$ 8,980
總堂數      3
已使用      1
預約佔用    0
可退款堂數  2
預計退款    NT$ 5,987
```

退款完成後：
- 已使用的 1 堂：保留（COMPLETED），不影響
- 剩餘的 2 堂：標記為作廢
- 顧客錢包 → 取消（沒可用堂數了）
- 報表淨收入扣除 5,987

## 四、為什麼系統會擋住退款

有時候你按 [退款]，畫面會跳出黃色提示，按鈕變灰按不下去。
**這不是 bug，是保護機制**。下面是常見原因：

### 「此方案仍有未完成預約，請先取消預約後再退款。」

**意思**：這個方案還有預約佔著堂數（顧客約了但還沒來，或日期沒到）。

**怎麼處理**：
1. 先進 [預約管理] 找到該顧客的待到店預約
2. 取消那些預約（堂數會退回方案）
3. 回來再退款

**為什麼系統要擋**：
如果允許直接退款，那筆預約會變成「沒堂數可扣，但已經約了」的孤兒狀態。
顧客如果到店了，店員會搞不清楚這筆要不要服務。

---

### 「此方案已有完成服務紀錄，不能全額退款。」

**意思**：你選了「全額退款」，但顧客已經來過至少一堂了。

**怎麼處理**：
- 把退款方式切換到「**退剩餘堂數**」
- 系統會自動算可退多少（已用的不退）

**為什麼系統要擋**：
全額退款代表「這筆交易完全不算了」。但顧客已經接受過服務、收過你的時間，
全額退會等於白白服務人家。系統自動擋下避免店長手滑。

---

### 「目前沒有可退款堂數。」

**意思**：這個方案的堂數**全部用完了**或**已經退過了**。

**怎麼處理**：
- 用完了 → 不能退了，請走人工流程
- 退過了 → 不需要重複退

**為什麼系統要擋**：
這是**防止重複退款**的最後一道關卡。如果沒有這個檢查，
有可能不小心退兩次，金額對不起來。

---

### 「此方案已有完成服務紀錄，如需退款請使用人工退款流程。」

**意思**：顧客已經完成所有堂數（全部 COMPLETED），但又要求退款。
v2 系統不支援自動退已完成的堂數。

**怎麼處理**：
- 跟主管討論退款金額
- 用後台「**手動調整**」功能建立負向交易（不在 Drawer 內）
- 或 v2.x 後續版本會加退已完成堂數的流程

---

## 五、退款之後報表怎麼看

退款完成後，你會在三個地方看到變化：

### 1. 交易紀錄列表
**會多一筆 REFUND 交易**（金額是負的，紅字顯示）：

```
2026-04-30  測試顧客  退款    -NT$ 5,987   ...
2026-04-29  測試顧客  課程購買  NT$ 8,980   ...
```

**重點**：原本那筆「課程購買 NT$ 8,980」**還在**，狀態仍是「已完成」。
這是設計上的：**讓帳永遠追得回來**。

### 2. 營收頁面（本月概況）

```
課程總收入   NT$ 8,980
退款        -NT$ 5,987
淨收入       NT$ 2,993
```

「課程總收入」維持原樣，「退款」獨立顯示，「淨收入」 = 兩者相加。

### 3. CSV 匯出

「店長月報」CSV 多一個「退款」欄位 + 「淨收」會自動扣掉退款。

```
店長  ...  課程總收入  退款       淨收     完成堂數
小明  ...  8,980      -5,987    2,993    1
```

## 六、最重要的觀念（請務必記住）

> ⚠️ **退款不是把原交易刪掉，也不是把原交易改成「已退款」狀態。**
>
> 退款是**新增一筆負數退款紀錄**，原交易維持不動。
>
> 這樣帳永遠追得回來：什麼時候賣的、什麼時候退的、退多少、誰退的、為什麼退，
> 全部都有紀錄，誰也改不了。

## 七、常見問題

### Q1. 我可以對同一筆交易退兩次嗎？
**不能。** 第一次退完之後，方案的堂數會被標記，再點退款會看到「目前沒有可退款堂數」。
這是防止意外重複退款。

### Q2. 退款後顧客會收到通知嗎？
v2 不會自動發訊息。請店長另外用 LINE / 電話告知顧客。

### Q3. 退款金額算錯了，怎麼辦？
**已經退了就不能改**。如果要修正，請：
1. 先在交易紀錄找到剛剛的 REFUND row
2. 截圖 + 記錄差額
3. 找主管討論
4. 走「手動調整」做差額補正

**不要試圖**：刪除退款 / 重複退款 / 直接改原交易金額。這些操作系統都會擋下來。

### Q4. 退款的錢實際上要怎麼還顧客？
系統只記**帳**，不會自動把錢匯出去。實際金流（現金 / 匯款 / 退到信用卡）
請店長依顧客原本付款方式處理，並在退款原因欄記錄做了什麼。

例如：
- 「現金退還，已給顧客 NT$ 5,987」
- 「匯款回顧客 OO 銀行末四碼 1234，2026-04-30 14:30」

### Q5. 我可以幫合作店長執行退款嗎？
**不行**，看店家權限設定。預設只有店長有「退款」權限，合作店長看不到退款按鈕。
如果合作店長需要退款，請店長代執行，或請總部開權限。

## 八、一張圖搞懂全流程

```
顧客要退款
    │
    ├─ 完全沒用過 + 沒預約 → 取消交易（v1 流程）✅
    │
    ├─ 完全沒用過 + 已預約 → 先取消預約 → 取消交易 ✅
    │                       或 → 先取消預約 → 退款（全額） ✅
    │
    ├─ 用過幾堂 + 想退剩下的 → 退款（剩餘堂數） ✅
    │
    ├─ 全部用完 + 還想退 → 找主管 → 手動調整 ⚠️
    │
    └─ 已退過 → 不能再退 ❌
```
